# Análisis de Incidente: Fuga de Datos y Mínimo Privilegio

Este repositorio contiene el análisis de un incidente de seguridad basado en el marco de trabajo NIST SP 800-53. El proyecto se enfoca en la evaluación de riesgos, la aplicación del principio de mínimo privilegio (Least Privilege) y la propuesta de controles de seguridad para prevenir filtraciones de datos.

## Escenario
El equipo de seguridad fue alertado sobre una filtración de documentos confidenciales en redes sociales. El incidente fue causado por un error humano tras otorgar permisos excesivos a un socio comercial externo durante una reunión de ventas.

##  Objetivos
- Analizar los factores que condujeron a la filtración.
- Revisar el control NIST AC-6 (Mínimo Privilegio).
- Proponer mejoras técnicas y operativas.
- Justificar las recomendaciones de seguridad.

##  Tecnologías y Marcos de Trabajo
- **NIST SP 800-53 Rev. 5** (Control AC-6).
- **NIST Cybersecurity Framework (CSF)**.
- **Markdown** para documentación técnica.
